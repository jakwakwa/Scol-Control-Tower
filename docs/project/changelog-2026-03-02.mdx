# Weekly Change Log: 2 March 2026

*StratCol Control Tower — 58 commits across 2 active branches this week*

## Breaking Changes

- **Schema updates**: New columns on `workflows` (`decisionType`, `targetResource`), `notifications` (`severity`, `groupKey`), and `ai_feedback_logs` (`relatedFailureEventId`). Database migration required before deploy.

## New Features

- **Decision routing decoupled from stages** — New `getDecisionEndpoint` helper resolves API endpoints from metadata instead of hardcoded stage numbers. Front-end and back-end no longer assume "stage 3 = procurement." (`43437cc`, Jaco Kotzee)
- **State-lock enforcement on all manual decisions** — Every human decision endpoint (risk-decision, quote approve/reject, final approval, contract review) now acquires a state lock, preventing ghost processes from overwriting finalized records (`3aea4d7`, `43437cc`, Jaco Kotzee)
- **Tiered notification system** — Low severity: log-only (no alert spam). Medium with groupKey: batched into a single summary card. High/critical: full `bg-destructive` dashboard + email alerts. Workflow table rows show bouncing alert icons linking to the graph view (`43437cc`, Jaco Kotzee)
- **Data lineage for failure resolution** — Manual decisions now link back to the specific automation failure via `relatedFailureEventId` on `ai_feedback_logs` (`43437cc`, Jaco Kotzee)
- **AI financial risk analysis** — Integrated AI-driven risk scoring into the Stage 3 analysis pipeline (`317a405`, Jaco Kotzee)
- **Sanctions screening with Firecrawl** — Enhanced sanctions with web-scraping evidence enrichment (`92c271c`, Jaco Kotzee)
- **Risk Review history** — "Show History" toggle surfaces past decisions alongside active reviews (`74e757b`, `a45b828`, Jaco Kotzee)
- **SA ID field + clickable applicant names** — PR #84 brought form improvements, scroll restoration, and notification auto-refresh (Copilot)
- **Workflow settings page** — New Slider, Switch, and Tooltip UI components (`2182a2c`, Jaco Kotzee)

## Bug Fixes
