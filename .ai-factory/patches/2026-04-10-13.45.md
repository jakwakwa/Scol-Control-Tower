# Identity verification: terminal write ordering and trusted-only terminal updates

**Date:** 2026-04-10 13:45 UTC
**Files:** `inngest/functions/services/id-verification.ts`, `lib/services/identity-verification-terminal.ts`, `db/schema.ts`, `app/api/onboarding/documents/[id]/route.ts`
**Severity:** critical

## Problem

`onFailure` could run `logWorkflowEvent` before persisting terminal verification status; when logging threw (e.g. FK or schema drift), `writeTerminalVerificationStatus` never ran and documents stayed `pending`.

Separately, placing `writeTerminalVerificationStatus` in a `"use server"` file would expose it as a client-callable action without auth; adding Clerk `auth()` inside would break Inngest callers that have no session.

## Root Cause

1. Ordering: fallible notification/event logging ran before the idempotent DB update that must succeed for operator-visible state.
2. Security/call surface: Server Actions are public RPCs; terminal verification writes belong on trusted server-only modules, not on exported actions.

## Solution

1. `**onFailure`:** Call `recordVendorCheckFailure`, then `await writeTerminalVerificationStatus(...)`, then `logWorkflowEvent` / `createWorkflowNotification` each chained with `.catch()` so logging cannot block or roll back the terminal write.
2. **Non-retriable path inside `step.run`:** Same order — terminal write first, then notifications with `.catch()`.
3. `**writeTerminalVerificationStatus`:** Implemented in `lib/services/identity-verification-terminal.ts` (no `"use server"`), with `[FIX]` logging and explicit error when DB client is missing.
4. **Schema / API:** Extended `document_uploads.verification_status` enum and onboarding document PUT `validStatuses` for `failed_ocr` and `failed_unprocessable`.

## Prevention

- For Inngest `onFailure`, treat workflow logging as best-effort: never await brittle logging before mandatory state commits.
- Do not export sensitive mutations from `"use server"` files unless every export performs auth; prefer shared `lib/` services for orchestration workers.
